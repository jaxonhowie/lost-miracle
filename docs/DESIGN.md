## 一、游戏概述

### 1.1 类型与目标

**静态图暗黑刷装 RPG + 半挂机战斗**。玩家在地牢中点击探索，遭遇怪物进行战斗，收集装备、强化装备、反复挑战精英与 Boss。

MVP 验证闭环：

```text
登录 → 选角色/新建 → 选地牢 → 探索 → 战斗 → 掉落 → 穿戴/强化 → 战力提升 → 继续刷怪
```

### 1.2 刻意不做（当前阶段）

- 大地图行走、攻击动画、复杂剧情
- 装备随机词条、装备品阶
- 地牢「通关」概念（Boss/精英可反复刷）
- 邮件/成就 UI、异步 PvP、交易行（在线能力见 §12）

### 1.3 技术栈

| 项目 | 说明 |
|------|------|
| 引擎 | Godot 4.6.2 stable |
| 语言 | GDScript |
| 分辨率 | 1280×720 |
| 表现 | ColorRect 纯色块 + Tween 动画 |

## 三、游戏流程

```text
启动
  ├─ 未登录 → LoginScene（注册 / 登录，JWT）
  └─ 已登录 → 主菜单（须联网，离线则阻断并提示重试）
       ├─ 固定展示 3 个角色槽（服务端列表，按最近登录排序）
       ├─ 点击已有角色 → GET 云存档 → 地牢探索
       └─ 点击「+ 创建新角色」→ POST 创建 → GET 默认档 → 地图选择
            └─ 地牢探索
                 ├─ 点击探索（随机事件）
                 ├─ 挑战精英（独立按钮，击杀后 2 分钟 CD）
                 ├─ 挑战 Boss（独立按钮，击杀后 5 分钟 CD）
                 ├─ 背包 Tab / 强化 R（底栏按钮）
                 └─ 战斗 → 结算 → 云同步 → 返回地牢
```

**地牢界面布局**（1280×720）：

- **顶栏**：等级；HP / MP / XP 进度条（条上覆盖当前/上限数值）；右上金币与当前地图名
- **中央**：探索日志、探索按钮、挑战精英 / Boss（含 CD 倒计时）
- **底栏**：「背包 (Tab)」「强化 (R)」两个独立按钮

- **无手动保存按钮**；战斗结算、事件确认、背包操作等时机**自动云同步**（`CloudSaveService.sync_to_cloud`）。
- 切场景、登出、退出游戏前须同步成功；失败则弹窗并阻止离开。
- 击杀 Boss 后**返回当前地牢**，不跳转选地图，无通关弹窗。

## 四、角色与属性

### 4.1 职业（代码已支持，当前仅开放战士）

| 职业 | ID | 主属性 | 副A | 副B |
|------|-----|--------|-----|-----|
| 战士 | warrior | STR | AGI | INT |
| 游侠 | ranger | AGI | STR | INT |
| 刺客 | assassin | AGI | STR | INT |
| 精灵 | elven | INT | STR | AGI |

新游戏当前固定 `warrior`，初始 STR=10, AGI=3, INT=3。

### 4.2 升级规则

- 经验需求：`level² × 50`
- 每级 +1 主属性
- 每 3 级循环额外 +1：**副A → 副B → 主**（`(level-1) % 3`）

### 4.3 进阶属性公式

| 属性 | 公式 |
|------|------|
| max_hp | 100 + STR × 5 |
| max_mp | 50 + INT × 10 |
| melee_atk | 10 + floor(STR / 3) |
| range_atk | 10 + floor(AGI / 3) |
| magic_atk | 10 + INT |
| def | floor(AGI / 3) |
| atk_spd | 1.0 + AGI × 0.01 |
| crit_rate | 0.05 + STR × 0.005 |
| crit_dmg | 1.5（基础） |
| dodge | 0.05 + AGI × 0.005 |
| hit | 1.0（基础） |

最终属性 = 基础属性 + 装备 base_stats（含品质倍率）+ 强化加值 + 套装 + 强化解锁特效 + 首饰属性 + 异系共鸣 + 祭坛临时 Buff。

## 五、战斗系统

### 5.1 模式

**实时战斗**（非回合制）：按攻速 `spd` 独立计时普攻，玩家可手动放技能。

攻击间隔：`2.0 / (spd × 0.15)` 秒。玩家攻速受 **战吼**（+20% `spd`）、套装、装备 `atk_spd` 等修正。

### 5.2 伤害公式

```text
基础伤害 = max(1, 攻击方 ATK - 防御方 DEF × 0.5)
暴击     = 基础伤害 × crit_dmg
闪避     = 伤害归零
技能     = 基础伤害 × 技能倍率 × (1 + skill_damage)
```

额外修正：亡灵增伤、Boss 增伤（套装）、受伤降低。

### 5.3 玩家技能（战士，快捷键 1/2/3）

| 技能 | 键 | MP | CD | 说明 |
|------|-----|-----|-----|------|
| 重击 | 1 | 10 | 10s | 150% 物理伤害 |
| 战吼 | 2 | 20 | 3s | **+20% 攻速**（作用于普攻间隔），持续 **300 秒**；**全局计时**（地牢探索 + 战斗中均递减）；已有 Buff 时不重复释放 |
| 血腥斩击 | 3 | 25 | 15s | 120% 伤害 + 10% 吸血 |

另有：生命药水 `[4]`、自动战斗切换。战吼状态存于 `PlayerData`（非战斗单位 Buff），生效时玩家头像上方显示 Buff 标识与剩余时间。

### 5.4 生命/魔法自然回复

- **全局**每 **3 秒** 跳一次（地牢探索 + 战斗中均生效），由 `PlayerData.tick_regen()` 驱动
- 每次回复：`HP + floor(STR/3)`，`MP + floor(INT/3)`（含装备主属性）
- 同帧递减战吼剩余时间（`battle_roar_remaining`）
- **无**战后 +50% HP/MP 恢复；胜利/战败均保留当前 HP/MP（战败 HP 先设为上限一半）
- `current_hp` / `current_mp` 不写入存档，加载云存档后由 `reset_for_new_game` / 进图逻辑重置；**战吼剩余时间与加成比例写入存档**

### 5.5 战斗表现

- 玩家：蓝色色块；怪物：红色/橙色/深红（按类型）
- 无 Emoji 头像，纯色块表示

### 5.6 战败 / 胜败后

- 胜利：发奖励，云同步，回地牢（自动战斗 3 秒后继续）；HP/MP 保持战斗结束时的值
- 战败：HP 设为 **一半上限**，回地牢；靠自然回复与药水恢复

## 八、强化系统（R2 风格）

### 8.1 材料

- **防具/武器**：仅消耗 **强化石**（普通 / 受祝福），**不消耗金币**
- **首饰**：仅消耗 **首饰强化石** / **受祝福首饰强化石**（见 §11.3），**不消耗金币**
- 受祝福强化石：成功率更高；失败时**有概率**保留装备（默认损毁率 15%，即 85% 保留）；首饰失败同样有概率损毁（见 §11.3）

### 8.2 成功率

| +N | 普通石 | 祝福石 |
|----|--------|--------|
| +0→+1 ~ +2→+3 | 100% | 100% |
| +3→+4 | 70% | 100% |
| +4→+5 | 50% | 90% |
| +5→+6 | 40% | 50% |
| +6→+7 | 30% | 45% |
| +7→+8 | 28% | 35% |
| +8→+9 | 25% | 33% |
| +9→+10 | 20% | 25% |

### 8.3 失败规则

- **失败不降级**
- +0→+1 ~ +2→+3：**100% 成功**，不会失败
- +3→+4 起可能失败；过 `safe_enhance_until`（默认 +3）后进入损毁区间
- 普通石失败：**35% 损毁**；受祝福石失败：**15% 损毁**（85% 保留，见 `enhance_rules.json`）
- 祝福装备在触发损毁判定时额外 **50%** 概率保住
- 武器 +1~+4 每级 ATK+1，+5 +2，+6~+10 每级 +3
- 防具 +1~+5 DEF/HP+1，+6~+7 +2，+8~+10 +3

### 8.4 全服隐性垫刀（在线，玩家不可见）

登录后强化由服务端权威掷骰。全服 Redis 池静默运作，**客户端无任何相关 UI 或文案**。

| 规则 | 说明 |
|------|------|
| 损毁计入 | 在线强化失败且装备损毁时，按被毁装备等级静默积累（`强化等级 + 1`） |
| 加成触发 | 池满阈值（默认 100）后，**下一次**在线强化额外 +25% 成功率（非必成），消耗阈值后重新积累 |
| 范围 | 防具/武器与首饰；须登录且联网，强化走服务端 API |

配置：`lost-miracle.enhance.padding.threshold` / `bonus-rate`  
API：`POST /characters/{id}/enhance/roll`（仅此一条对外接口）

## 九、掉落与经济

### 9.1 装备掉落（`LootManager`）

| 类型 | 掉装率 | 数量 | 金币 | 强化石率 |
|------|--------|------|------|----------|
| 普通 | 30% | 1 | 10~30 | 10% |
| 精英 | 60% | 1~2 | 30~80 | 25% |
| Boss | 100% | 2~3 | 100~300 | 50% |

另：50% 概率掉落 1~5 瓶生命药水。

**地牢差异化**（详见 §10.4–§10.6、`data/jewelry.json`）：

- **腐蚀沼泽**：不掉防具，改掉四系 +0 戒指（见 §10.4）
- **赤焰锻造厂**：不掉装备时仍掉金币；强化石改为首饰强化石 / 受祝福首饰强化石为主（见 §10.5）
- **永冻深渊**：不掉防具，改掉四系 +0 项链（见 §10.6）
- **荒骨墓穴**：上表通用规则

### 9.3 金币用途

金币来自战斗、宝箱等，用于**未来系统**（商店、修理等）。**强化（防具与首饰）均不消耗金币**。

|------|----------|------|

| Boss | 1 个槽位 | 击杀后 300 秒（5 分钟） |

API（需登录 + 角色归属校验）：

- `GET /characters/{id}/dungeons/{dungeonId}/spawns` — 查询槽位状态
- `POST .../spawns/encounter` `{ "type": "normal|elite|boss" }` — 占用槽位并返回怪物 ID
- `POST .../spawns/{slotId}/defeat` — 击杀，进入冷却
- `POST .../spawns/{slotId}/release` — 未击杀（玩家阵亡），释放占用

客户端 `SpawnService` 拉取状态；战斗胜利/失败分别调用 defeat / release。

### 11.1 定位

- 槽位：**`ring_left`（左手）+ `ring_right`（右手）**；**`necklace`（项链，单槽）**

## 十二、存档与在线

> 服务端 API 与架构见 **[`BACKEND_ARCHITECTURE.md`](BACKEND_ARCHITECTURE.md)**。

### 12.1 模型：云端唯一持久化

本游戏为**必须登录、必须联网**的在线游戏。游戏进度只存服务端，客户端内存中持有当前会话状态。

| 项目 | 说明 |
|------|------|
| 持久化 | MySQL `character_save` 表（整包 JSON + 版本号） |
| 客户端内存 | `SaveManager` 负责 `export_save_data()` / `import_save_data()`，**不写游戏存档到磁盘** |
| 本地文件 | 仅 `user://auth_token.json`（JWT 与当前角色 ID，不含游戏数据） |
| 联网门禁 | `ConnectivityMonitor` 探活；主菜单离线阻断；关键操作前检查可达性 |
| 主菜单 UI | 固定 **3 个槽位**，展示服务端角色列表（按最近登录排序）；不足 3 个时显示「+ 创建新角色」 |
| 槽位上限 | 每账号最多 3 个角色（服务端 `maxSlots`） |
| 角色管理 | 重命名（`PATCH /characters/{id}`）、删除（`DELETE`，含云端存档） |

**存档 JSON 内容**：玩家等级与属性、背包、装备（含 `ring_left` / `ring_right` / `necklace` 及旧 `ring` 迁移）、强化石与首饰强化石、药水、地牢进度与 Boss/精英 CD、当前地牢 ID、自动战斗开关、祭坛 Buff、战吼全局状态。

**不写入存档**：`current_hp` / `current_mp`（进图或读档后由逻辑重置）。

### 12.2 同步时机与策略

| 时机 | 行为 |
|------|------|
| 进入角色 | `GET /characters/{id}/save` 下载并 `import_save_data` |
| 战斗胜利结算 | `await sync_to_cloud` |
| 地牢事件确认 / 进战斗前 | `await sync_to_cloud` |
| 背包穿戴 / 卸下 / 丢弃 / 强化 | `await sync_to_cloud`（强化 roll 走服务端 API，成功后应用返回档） |
| 切换地牢 / 回主菜单 / 登出 / 退出 | `sync_before_scene_exit`，失败阻止离开 |
| 同步失败 | 内存重试队列 + 指数退避；网络恢复后自动冲刺 |

**版本冲突（409）**：弹窗选择「以云端为准」或「以本机为准」（force upload）。

**断网行为**：内存中可继续游玩，但切场景与登出会被同步拦住；进程强杀时未同步进度丢失（无本地兜底）。

### 12.3 已实现 / 待做

**已实现：**

- 注册 / 登录（JWT）、角色 CRUD（含重命名、删档）
- 云存档上下传、版本冲突、内存重试队列（`NetworkManager` + `CloudSaveService`）
- 网络探活与离线 UI（`ConnectivityMonitor` + `SyncStatusHud`）
- 战力排行榜查询
- 服务端权威强化 roll（`POST /characters/{id}/enhance/roll`）
- 邮件 / 成就客户端 UI，支持查看与领取奖励

**待做（P2+）：**

- 成就奖励改走邮件发放
- 槽位分页（角色数超出 UI 展示时）
- 异步 PvP、交易行
- 服务端权威战斗 / 掉落（防作弊，非当前阶段目标）
