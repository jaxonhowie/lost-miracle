# 1. MVP 定位

## 游戏暂定名 失落奇迹(Lost Miracle)

## 核心玩法闭环

```text
进入亡灵地牢1层
→ 击杀小怪
→ 获得金币 / 材料 / 装备
→ 装备更换或强化
→ 挑战精英怪
→ 获得稀有材料 / 高品质装备
→ 挑战Boss
→ 获得核心装备 / 强化突破材料
→ 重复刷图
```

## MVP 排除范围

多人联机、复杂剧情、职业系统、大地图、交易行、任务系统、宠物、技能树、背包高级排序、随机词缀深度系统。

核心优先：**打怪、掉落、装备、强化、刷新**。

| +1       | 20   | 强化石 x1       | 100%   |
| +2       | 40   | 强化石 x1       | 100%   |
| +3       | 80   | 强化石 x2       | 90%    |
| +4       | 150  | 强化石 x2       | 80%    |
| +5       | 250  | 强化石 x3       | 70%    |
| +6       | 400  | 中级强化石 x2   | 60%    |

| +8       | 900  | 中级强化石 x4   | 40%    |
| +9       | 1300 | 高级强化石 x2   | 30%    |
| +10      | 1800 | Boss强化核心 x1 | 25%    |

## 失败规则

```text
+1 到 +5：失败不降级
+6 到 +8：失败降1级
+9 到 +10：失败降1级，不爆装备
```

不做”装备损毁”。

## 强化属性成长

### 武器

```text
每强化 +1：攻击力 +10%
```

例：

```text
基础攻击 20
+1 = 22
+5 = 30
+10 = 40
```

### 防具

```text
每强化 +1：防御 +8%，生命 +5%
```

# 11. 掉落系统设计

## 掉落结构

数据表驱动，不硬编码。每条记录：`monster_id, item_id, min_count, max_count, drop_rate`。

## 装备掉落

```text
小怪：主要掉普通/优秀装备
精英：主要掉优秀/稀有装备
Boss：主要掉稀有/史诗装备
```

## 掉落表现

```text
怪物死亡
→ 掉落物从怪物位置弹出
→ 落地
→ 玩家靠近自动拾取
→ 屏幕左侧显示获得物品
```

# 12. 刷新系统设计

## 刷新点数据结构

每条记录：`spawn_id, monster_id, position, respawn_seconds, max_alive`。

## 刷新逻辑

```text
1. 地图启动时读取所有刷怪点
2. 如果怪物未存活，则检查死亡时间
3. 当前时间 - 死亡时间 >= 刷新时间，则尝试刷新
4. 如果玩家距离过近，则延迟刷新
5. 精英和Boss死亡时间写入存档
```

## 存档要求

小怪可不存死亡时间。精英和Boss需存：

```text
elite_1_dead_at
elite_2_dead_at
boss_dead_at
```

# 14. 战斗数值公式

## 伤害公式

```text
最终伤害 = max(1, 攻击方攻击 - 防御方防御)
```

暴击：

```text
如果触发暴击：
最终伤害 = 最终伤害 x 暴击伤害
```

## 玩家总属性

```text
玩家总攻击 = 基础攻击 + 武器攻击 + 强化攻击
玩家总防御 = 基础防御 + 防具防御 + 强化防御
玩家总生命 = 基础生命 + 装备生命 + 强化生命
```

# 17. 开发阶段与优先级

| 阶段 | 目标 | 关键内容 | 完成标准 |
|------|------|----------|----------|
| 1 | 基础手感 | Player.tscn、移动/跳跃/重力、摄像机、地图雏形、碰撞层、攻击 HitBox | 地图中移动、跳跃、攻击 |
| 2 | 怪物系统 | BaseMonster 状态机、3小怪、2精英、Boss | 玩家能击杀所有怪物 |
| 3 | 掉落背包 | ItemDatabase、DropTable、DropSystem、InventorySystem、背包 UI | 击杀后获得金币/材料/装备 |
| 4 | 装备系统 | EquipmentSystem、装备栏 UI、属性重算 | 穿装备后属性变化 |
| 5 | 强化系统 | EnhanceSystem、成功率/消耗/失败规则、强化 UI | +0 到 +10 可用 |
| 6 | 刷新存档 | SpawnSystem、SaveSystem、精英/Boss 时间持久化 | 重启游戏不能重复刷精英/Boss |

实现顺序：玩家移动攻击 → 地图碰撞 → 小怪战斗 → 掉落 → 背包 → 装备 → 强化 → 精英 → Boss → 刷新 → 存档 → UI 优化。

先让”小怪能刷、能死、能掉东西、能变强”跑通。
